# Lombard Perspectiva Monitor

Telegram-бот для мониторинга каталога [lombard-perspectiva.ru](https://lombard-perspectiva.ru). Отслеживает появление и исчезновение товаров, изменения цен и формирует аналитические отчёты для закупок.

## Возможности

- Сканирование всех категорий каталога:
  - швейцарские часы
  - ювелирные украшения
  - аксессуары
  - предметы искусства
- Фиксация в PostgreSQL: появление, уход с сайта, смена цены, резерв
- Автоотчёты настроенным получателям (ежедневно, еженедельно, ежемесячно)
- Команды по запросу: продажи, новинки, топ брендов, ценовые сегменты

## Требования

- Ubuntu Server (или другой Linux с Docker)
- [Docker](https://docs.docker.com/engine/install/) 24+
- [Docker Compose](https://docs.docker.com/compose/install/) v2
- Telegram-бот (токен от [@BotFather](https://t.me/BotFather))
- Telegram ID пользователей (узнать у [@userinfobot](https://t.me/userinfobot))

## Быстрый старт

### 1. Клонировать репозиторий

```bash
git clone https://github.com/daysema/lombard-perspectiva.git
cd lombard-perspectiva
```

### 2. Создать файл окружения

```bash
cp .env.example .env
nano .env
```

Заполните обязательные переменные:

| Переменная | Описание |
|------------|----------|
| `TELEGRAM_BOT_TOKEN` | Токен бота от BotFather |
| `TELEGRAM_ALLOWED_USER_IDS` | Кому слать автоотчёты (необязательно) |
| `POSTGRES_PASSWORD` | Надёжный пароль для БД |

Пример:

```env
TELEGRAM_BOT_TOKEN=7123456789:AAHxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
TELEGRAM_ALLOWED_USER_IDS=123456789,987654321
POSTGRES_PASSWORD=MyStr0ng_Passw0rd!
```

### 3. Запустить

```bash
docker compose up -d --build
```

При первом запуске:
- поднимется PostgreSQL;
- создадутся таблицы в БД;
- бот сразу выполнит полное сканирование каталога.

### 4. Проверить логи

```bash
docker compose logs -f bot
```

В Telegram напишите боту `/start` — должно прийти меню с командами.

## Команды бота

| Команда | Описание |
|---------|----------|
| `/start`, `/help` | Список команд |
| `/status` | Статус последнего сканирования |
| `/scan` | Запустить сканирование вручную |
| `/sold [дней]` | Товары, ушедшие с сайта (по умолчанию 7 дней) |
| `/new [дней]` | Новые поступления |
| `/top [дней]` | Топ брендов по «продажам» |
| `/fast [дней]` | Самые ходовые бренды (быстрее уходят с сайта) |
| `/price [дней]` | Распределение по ценовым сегментам |
| `/stats Бренд [дней]` | Статистика по бренду, например `/stats Rolex 30` |
| `/report [дней]` | Полная сводка за период |

## Расписание

По умолчанию (часовой пояс `Europe/Moscow`):

| Событие | Когда |
|---------|-------|
| Сканирование каталога | каждые 6 часов |
| Ежедневный отчёт | каждый день в 09:00 |
| Еженедельный отчёт | понедельник, 09:00 |
| Ежемесячный отчёт | 1-е число, 09:00 |

Настраивается в `.env`:

```env
SCAN_INTERVAL_HOURS=6
REPORT_DAILY_HOUR=9
REPORT_WEEKLY_DAY=0
REPORT_MONTHLY_DAY=1
TIMEZONE=Europe/Moscow
```

## Управление на сервере

```bash
# Остановить
docker compose down

# Перезапустить после изменения .env
docker compose up -d --build

# Обновить код
git pull
docker compose up -d --build

# Посмотреть состояние контейнеров
docker compose ps
```


## Как определяется «продажа»

Товар считается проданным, если он **пропал из активного каталога** на сайте. Товары в статусе «В резерве» остаются в каталоге и не считаются проданными. Для точной аналитики рекомендуется накопить данные минимум 1–2 недели после первого запуска.

## Безопасность

- Не коммитьте файл `.env` в репозиторий.
- Бот **открыт для всех** — любой пользователь Telegram может вызывать команды.
- Автоотчёты (ежедневные/еженедельные/ежемесячные) отправляются только пользователям из `TELEGRAM_ALLOWED_USER_IDS`. Если переменная пуста — автоотчёты отключены.
- PostgreSQL доступен только внутри Docker-сети, наружу порты не пробрасываются.
